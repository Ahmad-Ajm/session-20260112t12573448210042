# Clarification – FEAT-CV-CREATION

## 1. Executive Summary
ميزة إنشاء السيرة الذاتية تمكّن المستخدم المسجّل من إدخال بياناته الأساسية والمهنية في نموذج مبسّط، حفظها في النظام، وتحديثها لاحقًا، بحيث تصبح السيرة الذاتية جاهزة للعرض العام (مع ضوابط الخصوصية) وللاستخدام في البحث والتصدير.

## 2. Current Understanding
- **Problem / Opportunity**: صعوبة إنشاء سيرة ذاتية منظمة وجاهزة للمشاركة، وغياب منصة عربية بسيطة تجمع هذه السير وتعرضها.
- **Proposed Solution (Business View)**: توفير نموذج واحد واضح لإنشاء وتعديل السيرة الذاتية، مرتبط بحساب المستخدم، مع تخزين آمن للبيانات.
- **Main Users / Personas**: مستخدم عادي (باحث عن عمل).
- **Key Outcomes / KPIs**: عدد السير الذاتية المنشأة، معدل إكمال النموذج، عدد التعديلات الناجحة.

## 3. Scope & Boundaries
- In Scope:
  - إدخال البيانات الأساسية (الاسم، رقم الجوال، التعليم، الخبرات).
  - رفع صورة شخصية (حتى 2MB).
  - حفظ وتحديث السيرة الذاتية المرتبطة بالمستخدم.
- Out of Scope:
  - إنشاء أكثر من سيرة ذاتية لكل مستخدم.
  - دعم لغات متعددة داخل السيرة الذاتية.
- Assumptions:
  - لكل مستخدم سيرة ذاتية واحدة فقط.
  - المستخدم مسجّل الدخول قبل إنشاء أو تعديل السيرة.

## 4. Business Rules & Scenarios
- Business Rules:
  - لا يمكن حفظ السيرة بدون اسم وخبرة واحدة على الأقل.
  - حجم الصورة لا يتجاوز 2MB.
- Core User Journeys:
  - مستخدم يسجّل الدخول → يملأ نموذج السيرة → يحفظ → يعدّل لاحقًا.
- Edge / Error Cases:
  - رفع صورة بحجم أكبر من المسموح.
  - محاولة الوصول لسيرة مستخدم آخر للتعديل.

## 5. Non-Functional Expectations
- سهولة الاستخدام وبساطة النموذج.
- حفظ البيانات بشكل آمن.
- استجابة سريعة عند الحفظ والتحديث.
